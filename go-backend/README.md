# F1 Race Replay - Go Backend

Pragmatic Go rewrite of the FastAPI backend for 10x performance improvement on cache-hit scenarios (3–5s → 0.1–0.3s).

## Architecture

**Hybrid Go + Python:**
- **Go (Port 8000):** HTTP/WebSocket server, frame serving, frame generation algorithm
- **Python (Port 8001):** FastF1 telemetry extraction (subprocess, internal-only)

## Setup

### Prerequisites
- Go 1.22 or later
- Python 3.9+ (for FastF1 bridge)
- msgpack dependencies (auto-installed via go.mod)

### Build

```bash
cd go-backend
go mod download
go build -o f1-replay-go .
```

### Run

```bash
# Single command (starts both Go and Python services)
node dev.js

# Or separately:

# Terminal 1: Go backend on :8000
./f1-replay-go --port 8000 --cache-dir ../computed_data

# Terminal 2: Python FastF1 bridge on :8001 (Phase 2)
python3 ../scripts/fastf1_api.py

# Terminal 3: Frontend on :5173
cd ../frontend && npm run dev
```

## API Endpoints

### REST

- **GET /api/health** → `{"status": "ok"}`
- **POST /api/sessions** → Create new replay session (check cache first)
  - Request: `{"year": 2025, "round_num": 1, "session_type": "R", "refresh": false}`
  - Response: `{"session_id": "...", "status": "READY|LOADING", ...}`
- **GET /api/sessions/{session_id}** → Get session status
- **DELETE /api/sessions/cache** → Clear cache (optional pattern filter)

### WebSocket

- **ws://localhost:8000/ws/replay/{session_id}** → Frame streaming (Phase 2)
  - Upgrade triggers 60 Hz frame loop
  - Binary msgpack frames sent to client
  - Supports: play, pause, seek commands

## Performance Notes

- **Cache hit:** <500ms (vs 3–5s in Python)
- **Cache miss:** Python bridge subprocess
- **WebSocket:** 60 Hz frame streaming = 16.67ms per frame
- **Memory:** ~50–100 MB for full race session (154K frames)

## Troubleshooting

### "go: command not found"
Install Go 1.22+ from https://golang.org/dl

### "Cache file not found"
Ensure `../computed_data/*.msgpack` files exist (generated by Python backend or legacy system).

### WebSocket not connecting
- Check CORS middleware is enabled
- Verify `:8000` is not in use
- Check browser console for WebSocket errors
